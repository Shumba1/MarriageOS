# T-001 — Lock source authority and active set

## Objective
Establish the canonical authority layer, active feeder set, and reference-only sources for MarriageOS Phase 0.

## Why it matters
No source pipeline is reliable until builders and models know which files can overrule others, which files are active feeders, and which files are reference-only.

## Inputs
- `docs/MASTER_SPEC.md`
- `docs/DRIFT_REGISTER.md`
- `docs/README_ACTIVE_SET.md`
- `docs/SOURCE_MANIFEST.md`
- `docs/source-notes/*`
- current uploaded Marriage Project folder structure

## Worker role
Driver

## Output path
- `docs/README_ACTIVE_SET.md`
- `docs/SOURCE_MANIFEST.md`
- `docs/source-notes/authority-classification-notes.md`

## Output format
Markdown governance documents.

## Acceptance criteria
- canonical layer is explicit
- active feeder files are explicit
- reference-only files are explicit
- no current source item is left unclassified
- conflict rule is stated clearly
- canonical authority is easy to understand without reading the full archive

## Observability hooks
- canonical_sources_locked = true
- active_feeder_count
- reference_only_count
- unclassified_source_count
- source_authority_ready = true/false

## Status
inbox

## Notes
This is a gating task. Downstream source work should not proceed until source authority is locked.
