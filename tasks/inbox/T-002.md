# T-002 — Create normalized source records for currently available materials

## Objective
Create one normalized source record for each currently available major source item or source cluster in MarriageOS.

## Why it matters
Raw folders and long markdown files are not operationally usable until they are normalized into consistent source records.

## Inputs
- all currently available materials under:
  - `docs/`
  - `sources/raw/feeder-docs/`
  - `sources/raw/repo-ready-projection/`
  - `docs/pipeline/`
  - `docs/implementation/`
- `docs/pipeline/templates/raw-source-template.md`

## Worker role
Driver

## Output path
`docs/source-records/`

## Output format
One markdown file per source item or source cluster using a consistent source-record structure.

## Acceptance criteria
- each major source item has a normalized record
- each record includes source name, source tier, current role, allowed uses, forbidden uses, and summary
- records are consistent enough to be machine-browsable and human-readable
- no major source item is missing a record
- records reflect the authority rules from T-001

## Observability hooks
- raw_source_record_count
- unrecorded_source_count
- tier1_record_count
- tier2_record_count
- tier3_record_count
- source_record_coverage_percent
