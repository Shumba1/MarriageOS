# T-006 — Verify dossier readiness for downstream processing

## Objective
Evaluate whether the first dossiers are actually ready for downstream use or whether they are still thin, contradictory, under-specified, or overstuffed.

## Why it matters
The source pipeline is only successful if it produces dossier inputs that are trustworthy enough for later generation and implementation work.

## Inputs
- `sources/derived/dossiers/`
- `docs/contradictions.md`
- `docs/pipeline/prompts/05-verifier.md`

## Worker role
Critic

## Output path
`docs/dossier-readiness-report.md`

## Output format
Markdown report with one section per dossier:
- readiness status
- strengths
- missing inputs
- contradictions
- drift risk
- recommendation

## Acceptance criteria
- every dossier gets a readiness judgement
- unsupported claims are flagged
- thin dossiers are flagged
- contradictory dossiers are flagged
- next-step recommendations are concrete
- the report clearly distinguishes ready vs not-ready dossiers

## Observability hooks
- dossier_ready_count
- dossier_not_ready_count
- unsupported_claim_count
- missing_input_count
- downstream_ready_ratio
- verification_complete = true/false

## Status
inbox

## Notes
This is the gate before downstream generation or implementation planning begins.
